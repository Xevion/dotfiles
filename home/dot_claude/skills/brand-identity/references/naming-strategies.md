# Naming Strategies

Use these linguistic techniques when generating name candidates. Mix strategies to produce diverse candidates. Generate 30-50 candidates minimum before checking availability.

## Strategy Categories

### 1. Real Words (Repurposed)
Take an existing word and apply it to a new context.
- **Concrete nouns**: Rust, Flask, Spark, Crystal, Amber, Forge, Anvil, Harbor
- **Verbs**: Fetch, Render, Parse, Stitch, Weave, Glide
- **Nature**: Cedar, Birch, Heron, Falcon, Obsidian, Quartz
- **Mythology**: Atlas, Juno, Loki, Odin, Hermes

**Pros**: Memorable, easy to spell, existing mental associations
**Cons**: Hard to find available .com, may conflict with existing projects

### 2. Compound Words
Combine two short words.
- noun + noun: Starlight, Ironclad, Firewall, Cloudflare, Tailwind
- adj + noun: Fastlane, Quickdraw, Darkmatter, Redshift
- verb + noun: Breakpoint, Checkpoint, Logfire, Buildkit

**Pros**: Descriptive, memorable, often available
**Cons**: Can feel generic if not distinctive

### 3. Portmanteau (Word Blending)
Merge parts of two words.
- Kubernetes (cybernetes + helm concept)
- Instagram (instant + telegram)
- Pinterest (pin + interest)
- Technique: take the onset of word A + the coda of word B

**Pros**: Unique, brandable, distinctive
**Cons**: Can sound forced if not phonetically smooth

### 4. Truncation / Abbreviation
Shorten a longer word or phrase.
- Vercel (universal + cell? or acceleration)
- Deno (Node reversed/rearranged)
- Supabase (super + database)
- Technique: drop syllables, remove vowels, abbreviate

**Pros**: Short, modern feel
**Cons**: Meaning not always obvious

### 5. Invented Words (Neologisms)
Create a new word that sounds good.
- Xerox, Kodak, Hulu, Zillow
- Technique: use pleasant phoneme combinations, test for pronounceability
- Prefer: 2-3 syllables, alternating consonant-vowel patterns
- Avoid: hard consonant clusters (brkft), ambiguous spelling

**Pros**: Maximum availability, highly brandable, no conflicts
**Cons**: No inherent meaning, requires more marketing effort

### 6. Metaphor / Analogy
Name after something the project resembles conceptually.
- Docker (shipping containers)
- Kubernetes (helmsman)
- Apache (resilient, modular)
- Terraform (shaping land/infrastructure)

**Pros**: Rich associations, storytelling potential
**Cons**: Metaphor may not be obvious to everyone

### 7. Prefix/Suffix Patterns
Apply common tech prefixes or suffixes.
- Prefixes: un-, re-, de-, neo-, poly-, omni-, hyper-
- Suffixes: -ify, -ly, -io, -ium, -ize, -kit, -lab, -hub, -box, -flow

**Pros**: Familiar patterns, easy to generate many candidates
**Cons**: Can feel formulaic

### 8. Domain Hacks
Use the TLD as part of the word.
- del.icio.us, bit.ly, instagr.am, cr.yp.to
- Look for words that end in TLD strings: .io, .sh, .is, .im, .to, .so, .co, .me, .rs, .ai

**Pros**: Short URLs, clever, memorable
**Cons**: Dependent on specific TLD availability, harder to verbalize

## Phonetic Quality Checklist

For each candidate, mentally verify:
- [ ] **Pronounceable** — Can someone read it aloud correctly on first try?
- [ ] **Spellable** — Can someone type it after hearing it once?
- [ ] **Distinct** — Does it sound different from existing well-known projects?
- [ ] **No unfortunate meanings** — Check for unintended meanings in other languages or contexts
- [ ] **Works as a noun** — "I deployed it with [Name]" / "Check out [Name]"
- [ ] **Short** — Ideally 2-3 syllables, max 4
- [ ] **Domain-friendly** — No hyphens, numbers, or characters that complicate URLs
